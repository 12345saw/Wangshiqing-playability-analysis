# 如何检查图片路径（简单说明）

## 📋 验证方法（简单版）

### 步骤 1：打开开发者工具
1. 访问您的网站：https://12345saw.github.io/Wangshiqing-playability-analysis/
2. 按键盘上的 **F12** 键（或者右键点击页面，选择"检查"）

### 步骤 2：查看网络请求
1. 在开发者工具中，点击 **"Network"**（网络）标签
2. 刷新页面（按 F5 或点击浏览器的刷新按钮）

### 步骤 3：查看图片请求
1. 在 Network 标签中，您会看到很多请求
2. 找到图片相关的请求（通常文件名是 `.jpg`、`.png` 等）
3. 点击其中一个图片请求

### 步骤 4：查看图片的完整 URL
1. 在右侧的详情面板中，找到 **"Request URL"**（请求 URL）
2. 查看这个 URL 地址

### 步骤 5：确认 URL 是否正确
**正确的 URL 应该包含仓库名**，例如：
- ✅ 正确：`https://12345saw.github.io/Wangshiqing-playability-analysis/images/hero-bg.jpg`
- ❌ 错误：`https://12345saw.github.io/images/hero-bg.jpg`

如果 URL 中**没有** `/Wangshiqing-playability-analysis/`，说明 basePath 配置有问题。

## 🖼️ 更简单的方法（不需要开发者工具）

### 方法 1：直接测试图片链接
1. 在浏览器地址栏输入：
   ```
   https://12345saw.github.io/Wangshiqing-playability-analysis/images/hero-bg.jpg
   ```
2. 如果能看到图片，说明路径正确 ✅
3. 如果显示 404 错误，说明路径有问题 ❌

### 方法 2：查看页面源代码
1. 在网站上右键点击，选择"查看网页源代码"
2. 按 Ctrl+F 搜索 `hero-bg.jpg`
3. 查看图片路径是否包含 `/Wangshiqing-playability-analysis/`

## 🎯 总结

**第4步和第5步的意思是：**
- **第4步**：查看图片请求的完整网址（URL）
- **第5步**：确认这个网址中是否包含了仓库名 `/Wangshiqing-playability-analysis/`

如果包含，说明配置正确 ✅
如果不包含，说明需要修复 ❌
